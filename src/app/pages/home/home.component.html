D3.js Handbook

A practical guide for migrating Tableau charts to D3 — for teams building data-rich Angular applications.

What is D3?

D3 (Data-Driven Documents) is a JavaScript library for binding data to the DOM and applying data-driven transformations. It does not draw charts — it gives you the primitives to build exactly the chart you need, at any level of complexity.

Core philosophy: D3 is not a charting library. It is a data visualization toolkit. You describe the relationship between data and DOM — D3 handles the math.

D3 vs Other Libraries

Understanding where D3 sits in the ecosystem is essential before choosing it:

D3.js
Low-level
  • You own the rendering
  • Fully custom visuals
  • SVG / Canvas / HTML
  • Scales, layouts, generators
  • High code volume
  • No ceiling on complexity
Best for: custom, complex, branded charts
Chart.js
High-level
  • Config-driven API
  • Opinionated defaults
  • Canvas-based
  • Limited customization
  • Low code volume
  • Hits a ceiling fast
Best for: dashboards with standard charts
Highcharts
High-level
  • Declarative config object
  • Rich built-in chart types
  • Commercial license
  • Good accessibility
  • Moderate customization
  • Strong defaults
Best for: enterprise BI with standard needs
Tableau
GUI-driven
  • Drag-and-drop interface
  • No code required
  • Built-in data connectors
  • LOD expressions
  • Limited embed flexibility
  • Expensive at scale
Best for: ad-hoc analysis, non-dev users

D3's Core Philosophy

01
Selections — not components

D3 selects DOM elements and binds data to them. There is no "BarChart" class. You build it from rect elements joined to data arrays.

02
Scales — the heart of every chart

Scales map a data domain (e.g. 0–100) to a visual range (e.g. 0–500px). Master scaleLinear, scaleBand, scaleTime, scaleOrdinal and you can build anything.

03
Enter / Update / Exit pattern

When data changes, D3 reconciles DOM nodes: enter adds new, update modifies existing, exit removes stale. This is D3's version of diffing.

04
You write the tooltip

Unlike Tableau or Chart.js, D3 has no built-in tooltip. This is a feature — your tooltip can contain anything: sparklines, deltas, drill-down links, custom HTML.

05
Layouts compute — you render

D3 layouts (pie, stack, hierarchy, force) compute positions and angles. They never touch the DOM. You take those numbers and render however you choose.

When to choose D3 over a higher-level library

Choose D3 when you need:
✓ Custom tooltip with delta, drilldown, or rich content
✓ Multi-period overlays on the same axis
✓ Brush / zoom / pan interactions
✓ Animated transitions between data states
✓ Non-standard chart types (sankey, chord, treemap)
✓ Pixel-perfect branded visuals
✓ Full control over axes, grid, labels
Consider Chart.js / Highcharts when:
✗ Standard chart types are sufficient
✗ Rapid prototyping is the priority
✗ Team has no D3 experience and no time to learn
✗ Built-in responsive behavior is needed quickly
