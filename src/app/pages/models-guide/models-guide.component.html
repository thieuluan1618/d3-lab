Data Models Guide

A layered model architecture that cleanly separates API data, chart configuration, render-ready data, and tooltip state.

Data Flow Overview

API Response
MetricRecord[] / PeriodData[]
↓ ChartTransformService
Chart-Ready Data
ChartSeries[] + ChartPoint[]
↓ D3 Component
SVG Render
rect / path / circle
↓ hover events
Tooltip State
TooltipContext

Layer 1 — Raw / Domain Model

The shape of data from the API. Never mutate — pass into the transform service.

// What the API returns
interface MetricRecord {
  date: string;        // ISO string "2024-01-01"
  value: number;
  metadata?: Record<string, unknown>;
}

interface PeriodData {
  period: string;      // "2024-Q1", "2024"
  records: MetricRecord[];
}
Tableau equivalent: the raw data source rows before any calculation or filter.

Layer 2 — Chart Config Model

Describes how to render — not what data to show.

type ChartType = 'bar' | 'hbar' | 'hbar-group' | 'hbar-stack' | 'line' | 'pie' | 'area' | 'scatter';

interface ChartConfig {
  type: ChartType;
  title: string;
  subtitle?: string;

  // Field mapping
  xKey: string;                  // field → x-axis
  yKey: string;                  // field → y-axis

  // Axis config (Tableau: shelf settings)
  xAxis?: AxisConfig;           // label, format, scaleType, ticks, tickRotation
  yAxis?: AxisConfig;
  dualAxis?: DualAxisConfig;    // Tableau dual-axis worksheet

  // Color
  colorScheme?: string[];       // defaults to schemeTableau10
  colorKey?: string;            // field that drives color encoding

  // Dimensions
  margin?: ChartMargin;
  width?: number;
  height?: number;

  // Display toggles
  showGrid?: boolean;           // horizontal grid lines
  showValueLabels?: boolean;    // value text on bars / points
  showDataPoints?: boolean;     // dots on line charts
  legendPosition?: LegendPosition;

  // Interaction
  enableTooltip?: boolean;      // default true
  enableHighlight?: boolean;    // legend click → highlight
  enableBrush?: boolean;        // x-axis brush for zoom/filter
  enableZoom?: boolean;         // scroll to zoom

  // Chart-type-specific
  curveType?: 'linear' | 'monotoneX' | 'step' | 'basis';
  fillArea?: boolean;           // fill under line
  barPadding?: number;          // 0–1 spacing
  barRadius?: number;           // rounded bar tops
  innerRadius?: number;         // 0 = pie, >0 = donut
}
interface AxisConfig {
  label?: string;               // axis label text
  format?: string;              // d3-format: ',.0f', '.1%', '%b %Y'
  scaleType?: ScaleType;        // 'band' | 'point' | 'time' | 'linear' | 'log'
  ticks?: number;               // suggested tick count
  tickRotation?: number;        // degrees, e.g. -45
}

interface DualAxisConfig {
  yRightKey: string;            // field → right y-axis
  yRightLabel?: string;
  yRightFormat?: string;
}
Tableau equivalent: the Marks card + Columns/Rows shelf + Format pane + Dual-axis settings, serialized into one typed object.

Layer 3 — Chart Data Model

Transformed data ready for D3. One ChartSeries per line/bar-group. Multiple series = multi-period comparison.

interface ChartPoint {
  x: string | Date | number;
  y: number;
  label?: string;       // optional display label
  raw?: MetricRecord;   // back-reference for tooltip
}

interface ChartSeries {
  id: string;       // e.g. "revenue-2024"
  label: string;    // shown in legend
  period: string;   // drives multi-period toggle
  color?: string;
  points: ChartPoint[];
}
Tableau equivalent: the individual marks on the canvas — one per data point.

Layer 4 — Tooltip Model

Aggregates everything needed to render a rich tooltip. Populated on mouseover events.

interface TooltipContext {
  x: string | Date | number;
  series: TooltipSeries[];
  position: { left: number; top: number };
}

interface TooltipSeries {
  id: string;
  label: string;
  value: number;
  delta?: number;         // vs previous period
  deltaPercent?: number;  // vs previous period (%)
  color: string;
  raw?: MetricRecord;
}
Tableau equivalent: the Tooltip card — but fully custom. Can embed sparklines, links, delta badges.

Layer 5 — Chart View Model

Top-level model for a dashboard panel. Ties config, data, and UI state together.

interface ChartView {
  id: string;
  config: ChartConfig;
  periods: string[];         // all: ["2024", "2023"]
  series: ChartSeries[];     // all series (all periods)
  activePeriods: string[];   // user-toggled visible
  tooltip: TooltipContext | null;
}
Tableau equivalent: a single Tableau worksheet — config + data + view state in one place.

The Transform Layer

Never transform data inside a D3 component. Use ChartTransformService — keeps components as pure renderers.

// PeriodData[] → ChartSeries[]  (multi-period)
toMultiPeriodSeries(periods: PeriodData[]): ChartSeries[]

// Add delta vs previous period to tooltip
enrichWithDelta(current, prev?): TooltipSeries

// ChartPoint[] → % of total  (pie charts)
toPercentOfTotal(points): ChartPoint[]
Tableau equivalent: Calculated Fields + Table Calculations — but testable TypeScript functions.
